# ADR-006 — El Knowledge Engine no interpreta texto directamente

**Estado:** Aceptado

## Decisión

El Knowledge Engine primero traduce el texto de entrada al vocabulario interno del dominio
(Domain Vocabulary) y únicamente después ejecuta cualquier algoritmo de interpretación
(tokenizer, interpreter).

## Por qué

El vocabulario del dominio es una decisión de producto — cambia poco, describe cómo la
industria de serigrafía/textil nombra las cosas. El algoritmo de tokenización es una
decisión de implementación — probablemente se reescriba varias veces (reglas, IA, NLP). Si
el vocabulario se define primero e independiente del algoritmo, cualquier tokenizer futuro
puede reusarlo sin rediseñar nada.
